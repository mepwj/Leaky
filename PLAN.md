# Leaky - 가계부 앱 기획서

## 1. 프로젝트 개요

| 항목 | 내용 |
|------|------|
| 앱 이름 | Leaky (줄줄 새는 소비를 막아주는 가계부) |
| 플랫폼 | Android (React Native 0.84.0) |
| UI 프레임워크 | React Native Paper (Material Design 3) |
| 백엔드 DB | 자체 미니 PC 서버 (PostgreSQL 17 + Docker) |
| 데이터 입력 방식 | 수동 입력 (금융 API 연동 없음) |
| 참고 앱 | 위플 가계부, 뱅크샐러드, 토스, 편한가계부 |

> 금융기관 자동 연동은 공인인증서/마이데이터 사업자 등록 등 인증 절차가 필요하므로 제외.
> 수동 입력 중심으로 간편하고 직관적인 UX를 목표로 한다.

## 3. 사용자 플로우

### 3.1 회원가입 / 로그인

```
[앱 실행]
  ├─ Google OAuth 로그인 → 첫 로그인 시 → [사용자 정보 등록]
  ├─ 이메일 회원가입 → 이메일 인증 → [사용자 정보 등록]
  └─ 이메일 로그인 → [메인 화면]
```

#### 이메일 회원가입 플로우
1. 이메일 + 비밀번호 입력
2. 인증 이메일 발송
3. 이메일 인증 완료
4. 사용자 정보 등록 화면으로 이동

#### Google OAuth 플로우
1. Google 로그인 버튼 클릭
2. Google 계정 선택/인증
3. 첫 로그인 → 사용자 정보 등록 화면
4. 기존 사용자 → 메인 화면

### 3.2 사용자 정보 등록 (온보딩)

| 필드 | 필수 여부 | 설명 |
|------|-----------|------|
| 닉네임 (또는 이름) | 필수 | 앱 내 표시 이름 |
| 계좌 정보 | 선택 | 은행명, 별칭 (계좌번호 생략 가능) |
| 계좌 잔액 | 선택 | 현재 보유 금액 |
| 카드 정보 | 선택 | 카드사, 별칭 (카드번호 생략) |
| 카드 결제일 | 선택 | 매월 결제일 |
| 현금 보유액 | 선택 | 현금 잔액 |

- 계좌/카드는 복수 등록 가능
- 모든 정보는 이후 수정 가능

## 4. 화면 구성

### 4.1 화면 목록

| 화면 | 설명 |
|------|------|
| 스플래시 | 앱 로딩 화면 |
| 로그인 | 이메일 로그인 + Google OAuth |
| 회원가입 | 이메일/비밀번호 입력 |
| 이메일 인증 | 인증 대기 안내 |
| 온보딩 (정보 등록) | 닉네임, 계좌, 카드, 잔액 등록 |
| **메인 (캘린더)** | 월간 캘린더 + 일별 수입/지출 요약 |
| 일별 상세 | 해당 날짜 수입/지출 목록 |
| 수입/지출 등록 | 금액, 카테고리, 메모, 결제수단 입력 |
| 통계 | 월간/주간 카테고리별 소비 차트 |
| 자산 관리 | 계좌/카드 목록, 잔액 현황 |
| 설정 | 프로필 수정, 카테고리 관리, 로그아웃 |

### 4.2 하단 네비게이션 (Bottom Tab)

```
[ 홈(캘린더) ] [ 통계 ] [ + 등록 ] [ 자산 ] [ 설정 ]
```

### 4.3 메인 화면 (캘린더 뷰)

위플 가계부/편한가계부 참고:

- 상단: 월 선택 (< 2026년 2월 >)
- 중앙: 월간 캘린더
  - 각 날짜 셀에 수입(파란색)/지출(빨간색) 합계 표시
  - 오늘 날짜 하이라이트
- 하단: 월간 요약
  - 총 수입 / 총 지출 / 잔액
- 날짜 터치 → 일별 상세 화면

### 4.4 수입/지출 등록 화면

| 필드 | 설명 |
|------|------|
| 구분 | 수입 / 지출 토글 |
| 금액 | 숫자 키패드 입력 |
| 날짜 | 기본 오늘, 변경 가능 |
| 카테고리 | 목록에서 선택 |
| 결제수단 | 현금 / 계좌 / 카드 선택 |
| 메모 | 자유 텍스트 (선택) |

## 9. 개발 우선순위 (Phase)

### Phase 1 - MVP (핵심 기능)

1. 프로젝트 세팅 (React Native Paper, Navigation 등)
2. 로그인/회원가입 (Google OAuth + 이메일)
3. 이메일 인증
4. 온보딩 (사용자 정보 등록)
5. 메인 캘린더 화면
6. 수입/지출 등록 (카테고리, 메모, 결제수단)
7. 일별 상세 내역
8. 기본 자산 관리 (계좌/카드 CRUD)

### Phase 2 - 통계 및 분석

1. 월간/주간 통계 차트
2. 카테고리별 분석
3. 전월 비교
4. 예산 설정 및 관리

### Phase 3 - 편의 기능

1. 반복 지출 자동 등록
2. 자주 쓰는 내역 즐겨찾기
3. 검색 기능
4. 커스텀 카테고리 관리
5. 카드 결제 예정액

### Phase 4 - 확장 (향후)

1. 엑셀 내보내기
2. 주간/월간 소비 리포트
3. 푸시 알림 (예산 초과, 소비 피드백)
4. 다중 화폐
5. 다크 모드

## 10. 데이터 모델 (초안)

### User
```
- id
- email
- nickname
- provider (google / email)
- created_at
- updated_at
```

### Account (계좌)
```
- id
- user_id
- bank_name (은행명)
- alias (별칭)
- balance (잔액)
- created_at
- updated_at
```

### Card (카드)
```
- id
- user_id
- card_company (카드사)
- alias (별칭)
- payment_day (결제일)
- created_at
- updated_at
```

### Transaction (수입/지출)
```
- id
- user_id
- type (income / expense)
- amount (금액)
- category_id
- payment_method (cash / account_id / card_id)
- memo
- date
- created_at
- updated_at
```

### Category (카테고리)
```
- id
- user_id (null이면 기본 제공)
- type (income / expense)
- name
- icon
- sort_order
- is_default
```

### Budget (예산)
```
- id
- user_id
- category_id (null이면 전체 예산)
- amount
- month (YYYY-MM)
```

## 11. 서버 인프라

### 서버 정보

| 항목 | 내용 |
|------|------|
| 호스트 | mepwj.iptime.org |
| OS | Ubuntu 24.04.3 LTS |
| Docker | 28.3.3 + Compose 2.39.1 |
| Node.js | 22.18.0 |
| DB | PostgreSQL 17.8 (Docker, leaky-db 컨테이너) |
| DB 이름 | leaky |
| DB 포트 | 5432 (외부 포트포워딩 완료) |

### DB 관리: Prisma

- `prisma/schema.prisma`에 데이터 모델 정의
- `npx prisma migrate dev` → 테이블 자동 생성/변경
- 마이그레이션 히스토리가 git으로 관리됨
- TypeScript 타입 자동 생성 (프론트와 타입 공유 가능)

### API 서버 (구성 예정)

- Node.js + Express 또는 Fastify
- Prisma ORM으로 DB 접근
- Docker 컨테이너로 배포 (leaky-api)
- 포트: 3000
